The Fundamental Problem: Vectors Are Dead Artifacts

Today's embeddings are frozen at creation. They don't know when they're stale, can't assess their own reliability, and have no awareness of how they relate to other knowledge. VersionRAG research shows standard RAG achieves only 58% accuracy on versioned queries. Our approach targets 90%+.

Today: Dumb Vector

A fixed array of 768 floats. Encodes semantic meaning at one frozen moment. No concept of time, confidence, or relationships. When source data changes, it doesn't know. Confidently wrong forever.

float[768] — that's it. No lifecycle, no self-awareness.

Proposed: Smart Vector

An embedding enriched with temporal awareness, confidence decay, and relational edges. Knows when it was valid, how confident it should be right now, and which vectors depend on it. Adapts or fades when the world changes.

semantic[768] + temporal[64] + confidence engine + graph edges

Three Properties That Make a Vector "Smart"

1

Temporal Awareness

Time is encoded in the vector space itself, not just as external metadata. Queries naturally surface recent, temporally valid results. Vectors with expired validity windows score near-zero.

Research: DE-SimplE, ATiSE, PTBox — temporal KG embeddings. LiveVectorLake (2026) for streaming temporal retrieval.
2

Confidence Decay

Each vector carries a confidence score that decays over time (Ebbinghaus forgetting curve) but strengthens when users validate answers. Creates a self-tuning feedback loop. Dormant vectors are deprioritized, not deleted.

Research: Neuroscience memory reconsolidation. MAGMA dual-stream consolidation. KDD 2025 uncertainty quantification survey.
3

Relational Awareness

Vectors maintain graph edges to dependents. When one changes, a ripple propagates — like GNN message passing. The "sky is blue" update triggers review of "atmospheric model uses blue baseline."

Research: GNN message passing. MAGMA multi-graph memory. A-MEM agentic memory (2025).

Five-Stage Lifecycle (Inspired by Neuroscience)

Mirrors the brain's memory consolidation: hippocampal fast-write → sleep-time replay → reconsolidation → forgetting → schema update.

1

Encoding (Hippocampal Fast-Write)

Status: UNCONSOLIDATED | Available immediately

New text is chunked with source-offset tracking, embedded, and inserted with status=UNCONSOLIDATED. Queryable immediately but not cross-referenced. Old versions auto-deprecated via doc_id+chunk_index matching. Relationship edges transferred from old→new.

2

Consolidation (Sleep-Time Replay)

Background cron job | Status: → ACTIVE

ConsolidationAgent runs periodically. Recalculates confidence for all vectors. Detects contradictions (same topic, different content). Builds relationship graph (depends_on, depended_by). Propagates ripples from recent changes. Marks unconsolidated→active.

3

Retrieval & Reinforcement (Reconsolidation)

Every query modifies the vector's confidence

4-signal scoring: similarity + temporal + confidence + relational. When user accepts answer → positive_feedback++ (confidence boost). When user corrects → negative_feedback++ (confidence penalty). Every access slightly modifies the vector, just like memory reconsolidation.

4

Decay & Forgetting

Status: → DORMANT when confidence < 15%

Exponential decay: C(t) = C₀ × 2^(-t/half_life). Vectors never accessed gradually lose confidence. Below threshold → DORMANT (still in DB, deprioritized). 180-day-old vector with no reinforcement → confidence drops to ~1%.

5

Supersession & Archival

Status: → DEPRECATED → ARCHIVED

When new version arrives, old is DEPRECATED. Confidence, edges, and relationships transfer to the new vector. Old vector archived for audit trail — not deleted. Knowledge evolves gracefully. Ripple propagator notifies all dependents of the change.

Implementation: ConfidenceEngine

The core algorithm that makes vectors self-aware of their own reliability. Combines time decay, user feedback, and access reinforcement.

Confidence Formula

Three factors combine to produce the current confidence at any moment in time:

# Step 1: Exponential time decay (Ebbinghaus forgetting curve)
decayed = base_confidence × 2^(-age_days / half_life_days)

# Step 2: User feedback adjustment (memory reconsolidation)
adjusted = decayed + (positive_count × 0.03) - (negative_count × 0.08)

# Step 3: Access reinforcement (frequently recalled = stronger)
final = adjusted + log(access_count + 1) × 0.01

# Result clamped to [0.01, 1.0]
Full Implementation
class ConfidenceEngine:

    def __init__(self,
                 half_life_days=30.0,           # Confidence halves every 30 days
                 feedback_positive_boost=0.03,   # +3% per thumbs-up
                 feedback_negative_penalty=0.08, # -8% per correction
                 access_reinforcement=0.01,      # Small boost per access
                 dormant_threshold=0.15):        # Below 15% → dormant
        ...

    def compute_decay(self, base_confidence, age_days):
        """Exponential decay: C(t) = C₀ × 2^(-t / half_life)"""
        return base_confidence * (2 ** (-age_days / self.half_life_days))

    def apply_feedback(self, confidence, pos_count, neg_count):
        """Positive feedback strengthens, negative weakens."""
        boost = pos_count * self.feedback_positive_boost
        penalty = neg_count * self.feedback_negative_penalty
        return clamp(confidence + boost - penalty, 0.01, 1.0)

    def apply_access_reinforcement(self, confidence, access_count):
        """Diminishing returns: log(accesses) × 0.01"""
        reinforcement = math.log(access_count + 1) * self.access_reinforcement
        return min(1.0, confidence + reinforcement)

    def compute_current_confidence(self, vector, reference_time):
        """Full pipeline: decay → feedback → reinforcement"""
        age = (reference_time - vector.created_at).days
        decayed  = self.compute_decay(vector.base_confidence, age)
        adjusted = self.apply_feedback(decayed, vector.positive_feedback,
                                                    vector.negative_feedback)
        final    = self.apply_access_reinforcement(adjusted, vector.access_count)
        return round(final, 4)

    def should_go_dormant(self, confidence):
        return confidence < self.dormant_threshold

ConflictDetector — Finds Contradictions

Groups vectors by topic similarity, then checks if high-topic-overlap pairs have low content similarity (same subject, different claims = contradiction).

class ConflictDetector:

    def detect_conflicts(self, vectors, topic_threshold=0.3,
                         content_threshold=0.4):
        """
        High topic overlap + Low content similarity = CONTRADICTION
        High topic overlap + High content similarity = REDUNDANCY
        """
        for va, vb in all_active_pairs:
            topic_sim   = jaccard(va.content, vb.content)
            content_sim = sequence_similarity(va.content, vb.content)

            if topic_sim > 0.3 and content_sim < 0.4:
                # Contradiction! Resolve by: recency → authority → feedback
                winner = self._resolve_conflict(va, vb)

    def _resolve_conflict(self, va, vb):
        """Majority vote: recency_winner + authority_winner + feedback_winner"""
        recency_winner   = va if va.created_at > vb.created_at else vb
        authority_winner  = va if va.base_confidence > vb.base_confidence else vb
        feedback_winner   = va if va.feedback_ratio > vb.feedback_ratio else vb
        return majority_vote([recency_winner, authority_winner, feedback_winner])
RipplePropagator — GNN-Inspired Update Propagation

When a vector changes, BFS traversal notifies all dependents within max_depth=2 hops. Each hop reduces the confidence penalty (signal attenuates with distance).

class RipplePropagator:

    def __init__(self, penalty_per_hop=0.15, max_depth=2):
        ...

    def propagate(self, changed_vector_id, all_vectors):
        """BFS from changed vector through depended_by edges."""
        visited = set()
        self._propagate_recursive(changed_vector_id, depth=0)

    def _propagate_recursive(self, vector_id, depth):
        if depth >= self.max_depth:
            return  # Stop cascade

        for dep_id in vector.depended_by:
            # Signal attenuates: penalty × (1 / (depth+1))
            penalty = 0.15 * (1.0 / (depth + 1))
            dependent.base_confidence -= penalty
            dependent.contradictions.append(changed_id)  # Flag for review

            # Continue to next hop
            self._propagate_recursive(dep_id, depth + 1)

Consolidation Orchestration — Full Cycle
class ConsolidationAgent:

    def run_consolidation(self, vectors, recent_changes):
        """Run one full consolidation cycle (call via cron)."""

        # 1. Recalculate confidence for all active vectors
        for vec in active_vectors:
            new_conf = confidence_engine.compute_current_confidence(vec)
            if new_conf < 0.15:
                vec.status = DORMANT

        # 2. Detect contradictions between vectors
        conflicts = conflict_detector.detect_conflicts(vectors)
        for conflict in conflicts:
            mark_contradictions(conflict.va, conflict.vb)

        # 3. Build relationship graph edges
        relationships = relationship_builder.build_relationships(vectors)
        relationship_builder.apply_relationships(vectors, relationships)

        # 4. Propagate ripples from recent changes
        for changed_id in recent_changes:
            ripple_propagator.propagate(changed_id, vectors)

        # 5. Promote unconsolidated → active
        for vec in unconsolidated:
            vec.status = ACTIVE

Implementation: SmartVectorDB Retrieval Pipeline

The 4-signal scoring formula that replaces naive cosine similarity.

Retrieval Scoring Formula
final_score = 0.35 × similarity       # How relevant to the query
            + 0.25 × temporal_score   # How fresh (exponential decay)
            + 0.25 × confidence       # How trusted (decay + feedback)
            + 0.15 × relational_bonus # How well-connected

Each signal is computed independently:

Signal 1: Semantic Similarity
# POC: keyword overlap. Production: cosine(embed(query), embed(chunk))
similarity = len(query_words ∩ content_words) / len(query_words)
Signal 2: Temporal Score (with Validity Windows)
def temporal_score(vector, reference_time):
    # Check validity window first
    if vector.temporal_validity_end and reference_time > validity_end:
        return 0.05  # Expired knowledge → near-zero

    # Exponential decay with 30-day half-life
    age_days = (reference_time - vector.created_at).days
    return 2 ** (-age_days / 30)
Signal 3: Confidence (Live Computation)
# Calls ConfidenceEngine.compute_current_confidence(vector)
# Combines: time_decay × feedback_adjustment × access_reinforcement
confidence = confidence_engine.compute_current_confidence(vector)
Signal 4: Relational Bonus
def relational_bonus(vector):
    """Well-connected vectors are more integrated/reliable."""
    if vector.edge_count == 0:
        return 0.0
    return min(0.3, math.log(1 + vector.edge_count) * 0.1)
